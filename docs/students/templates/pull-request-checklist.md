# 📤 Pull Request Checklist (Student Version)

> Run through this *before* you open your Pull Request. It's the same stuff the
> robots and your mentor will check — do it yourself first and your PR will sail
> through! ⛵

---

## Before I open my PR

**I planned it**
- [ ] I know who this is for and what it should do (I have a [Feature Spec](./feature-spec.md)).

**I built it**
- [ ] My change does what I planned.
- [ ] I made it on a branch (not on `main`).
- [ ] My branch name describes the work (e.g. `feat/join-button`).

**I checked it myself**
- [ ] I ran `npm run lint` → it passed ✅
- [ ] I ran `npm test` → it passed ✅
- [ ] I ran `npm run dev` and clicked through my feature in the browser.
- [ ] I tried to break it (empty inputs, fast clicks, weird text).
- [ ] I checked it on a phone-size screen.

**I saved it properly**
- [ ] My commit messages describe what I did (not "stuff" or "asdf").
- [ ] I pushed my branch to GitHub.

---

## When I open the PR

- [ ] I wrote a clear title (e.g. "Add a Join button to the homepage").
- [ ] I filled out the PR description (what I changed and why).
- [ ] I linked the issue if there is one (e.g. "Closes #12").

## After I open the PR

- [ ] CI shows a green check ✅ (if red, I clicked "Details," fixed it, and pushed again).
- [ ] I replied to my mentor's comments and clicked **"Resolve conversation"** on each.
- [ ] I kept my branch up to date with `main`.

---

✅ **All checked? You're ready. Request a review and high-five yourself!** 🙌
